Autodesk Construction Cloud Data Connector

Overview

A common feature request for Autodesk's Construction Cloud solution has been the ability to connect Business Intelligence (BI) tools to analyze and process the data created in the various offerings. The Data Connector is the answer for this requirement. The Data Connector allows users to request and download data housed in the Construction Data Platform in a well defined schema/format. This data can then be used for further analysis and to drive customer specific workflows.

Data Schema & Design

A central aspect to the Data Connector is the data schema for data extracts. The data is extracted into a well defined schema much in the way a traditional relational database schema is designed and we refer to this as the ACC Data Schema. Each service group is a separate "database" schema design and each individual table in this design will be delivered via separate CSV file extracts. When a data request is completed, the Data Connector will generate a ZIP archive file that will be made available for download by the user. The naming conventions and directory structure for ZIP file extracts are the following:
  • autodesk_data_extract.zip
    • <schema1>_<table_name1>.csv
    • <schema1>_<table_name2>.csv
    • <schema1>_<table_name3>.csv
    • <schema2>_<table_name1>.csv
    • <schema2>_<table_name2>.csv
    • <schema2>_<table_name3>.csv
    • etc...
    • docs <directory>
      • README.html - this document
      • schemas <directory>
        • <schema>.html - detailed schema documentation for a particular schema
        • <schema>.json - JSON data dictionary for the particular schema
        • schema.json - JSON data dictionary file that describes the the entire ACC Data Schema definition
        • schema_create_mssql.sql - SQL file that will create the ACC Data Schema in a Microsoft SQL Server database
        • schema_create_postgres.sql - SQL file that will create the ACC Data Schema in a Postgres database
        • activities_<schema>_verb_column_details.json - programmatic (JSON) description of the Activities data for the schema
        • activities_<schema>_verb_column_details.html - detailed schema documentation for Activities data for the schema
    • metadata.csv - a file containing simple metadata on this extraction. The columns for this file are:
      • created_at - the timestamp for the extraction creation
      • region - the region where this data exists. Possible values: US: United States / EMEA: Europe/ AUS: Australia

Column Definition Types

The ACC Data Schema tables are often delivered in a "lowest common denominator" format (i.e. CSV) and as a result the column type definitions are limited to the following:
  • boolean
  • date: string
  • enum: string
  • number
  • string
  • string: UUID (xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx)
  • string: null
  • timestamp: SQL (YYYY-MM-DD HH:MM:SS.mmm) (UTC)
